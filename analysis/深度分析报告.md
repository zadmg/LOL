```

├── 资源段 (.rsrc, 68KB) — 窗口/图标/对话框/Manifest

          ↓ 硬件指纹解密
    ENHRmteD.dat (MJTVPFeD.dat, 113,664字节)
    ├── [0x00000-0x0B9F0] 外层 EXE (PE32+, 控制台程序)
    ├── [0x0B9F0-0x0D3F0] PE1: dbgcore.dll (UPX → 解压 10,240B)

```

## 4. 反检测体系

### 4.1 六层防御架构

```
第1层: 文件伪装与加密
  ├── VMProtect 标记壳 (保护登录函数, 干扰静态分析)
  ├── RC4 加密 overlay (密钥=硬件指纹, 无法跨机器解密)
  ├── .dat 扩展名伪装 (实际是 PE 可执行文件)
  └── UPX 双层加壳 (dbgcore.dll + flyinput.dll)

第2层: DLL 劫持注入 (核心机制)
  ├── 伪装系统 dbgcore.dll (Windows 崩溃转储组件)
  ├── 利用 Windows DLL 搜索顺序 (游戏目录优先)
  ├── 代理转发所有导出函数 (MiniDumpWriteDump/ReadDumpStream)
  └── 游戏无感知加载 (DllMain 中 LoadLibrary flyinput.dll)

第3层: 反调试
  ├── NtSetInformationThread(ThreadHideFromDebugger, 0x11)
  ├── 动态 GetProcAddress (绕过 IAT 分析)
  └── FNV-1a 哈希反字符串 (阻止 strings 命令提取)

第4层: 运行时行为隐藏
  ├── 无 ReadProcessMemory/WriteProcessMemory (进程内直接读写)
  ├── 无新线程创建 (利用 DllMain 初始化)
  ├── 无独立窗口 (不创建 overlay, 不 hook D3D)
  └── SendInput 模拟输入 (1ms 间隔, 模拟正常操作)

第5层: 网络通信加密
  ├── Pikachu 自建 CA + 证书固定 (中间人攻击无效)
  ├── 签名协议: 123[data]456[key]789
  └── 硬件指纹绑定

第6层: 自清理
  ├── 外层 EXE 自删除 (std::filesystem::remove_all)
  └── 释放到 C:\Users\Public\FlyPY\ (非敏感目录)
```

### 4.2 DLL 劫持详细流程

```
游戏启动 → 加载 dbgcore.dll
  ↓
Windows DLL 搜索顺序: 游戏目录 → System32
  ↓ (假 dbgcore.dll 已提前释放到游戏目录)
加载假 dbgcore.dll
  ↓
假 dbgcore.dll DllMain:
  ├── DisableThreadLibraryCalls   ← 减少线程通知痕迹
  ├── GetSystemDirectoryA          ← 获取 System32 路径
  ├── LoadLibrary(真 dbgcore.dll)  ← 加载真 DLL
  ├── GetProcAddress(MiniDumpWriteDump)  ← 保存真函数指针
  ├── GetProcAddress(MiniDumpReadDumpStream)
  └── LoadLibrary(flyinput.dll)    ← 加载辅助 DLL
  ↓
MiniDumpWriteDump 导出:
  jmp [真函数指针]  ← 完美转发, 游戏无感知
```

#### 5.3.8 功能开关位 [base+0x14A0] 语义解密

通过 Grep 全 .text 段搜索 test byte ptr [rxx+14A0h], mask 指令(共 91 处),解密出 3 个 bit 的实际语义:

| bit | mask | 出现次数 | 语义 | 代码模式 |
|-----|------|---------|------|---------|
| **bit 0** | 0x01 | 32 次 | **主自动瞄准开关** | 函数入口检查,未启用则整个函数跳过 |
| **bit 2** | 0x04 | 28 次 | **右键取消模式** | 检测鼠标右键是否取消当前施法 |
| **bit 3** | 0x08 | 30 次 | **SendInput 执行许可** | 决定是否真正发送输入(否则仅计算不执行) |

逻辑关系:
- 执行辅助 = bit0(主开关) AND (NOT bit2(右键取消) OR 右键未按下)
- 实际发送输入 = bit0 AND bit3(SendInput 许可) AND (NOT bit2 OR ...)

sub_18000CE80 缺少 bit2 检查,说明该英雄不支持右键取消(推测为持续施法类英雄)。

public static class Dash
{
    private static readonly Dictionary<int, DashItem> DetectedDashes;

    public static DashItem GetDashInfo(this Obj_AI_Base unit)
    {
        return DetectedDashes.ContainsKey(unit.NetworkId)
            ? DetectedDashes[unit.NetworkId] : new DashItem();
    }

    public static bool IsDashing(this Obj_AI_Base unit)
    {
        if (DetectedDashes.ContainsKey(unit.NetworkId) && unit.Path.Length != 0)
            return DetectedDashes[unit.NetworkId].EndTick != 0;
        return false;
    }

    // 监听 OnNewPath 事件检测位移
    private static void ObjAiHeroOnOnNewPath(Obj_AI_Base sender, GameObjectNewPathEventArgs args)
    {
        if (sender.IsValid<Obj_AI_Hero>())
        {
            if (args.IsDash)
            {
                // 记录位移信息: StartTick/Speed/StartPos/Path/EndPos/EndTick/Duration
                DetectedDashes[sender.NetworkId].EndTick =
                    DetectedDashes[sender.NetworkId].StartTick
                    + (int)(1000 * (EndPos.Distance(StartPos) / Speed));
            }
        }
    }

    public class DashItem
    {
        public int Duration;
        public Vector2 EndPos;
        public int EndTick;
        public bool IsBlink;
        public List<Vector2> Path;
        public float Speed;
        public Vector2 StartPos;
        public int StartTick;
        public Obj_AI_Base Unit;
    }
}

## 8. 加密与混淆技术

### 8.1 加密层次

| 层次 | 算法 | 密钥 | 保护对象 |
|------|------|------|----------|
| 外层 | VMProtect 标记壳 | - | 16.13.exe 登录函数 |
| 中层 | UPX 压缩 | - | dbgcore.dll + flyinput.dll |
| 内层 | RC4 流密码 | CPUInfo + HDInfo + ENode | Overlay (MJTVPFeD.dat) |
| 通信 | 签名协议 | 3 个 Token | API 通信 |
| 字符串 | FNV-1a 哈希 | - | 英雄名/技能名/配置键 |

### 8.2 RC4 密钥生成

```
密钥 = 硬件指纹组合
  ├── CPUInfo  (CPUID 指令 + 注册表 HKLM\HARDWARE\DESCRIPTION\System\BIOS)
  ├── HDInfo   (DeviceIoControl(\\.\PhysicalDrive0) + GetVolumeInformation)
  └── ENode    (GetAdaptersInfo → MAC 地址)
       ↓
  MD5 哈希 (CALG_MD5 = 0x8003)
       ↓
  RC4 密钥 (推测 16 字节)
```

**密钥流特征**: 熵值 7.9983, 256 唯一值, 无重复模式, 无周期。
